## 🚀 Misión de Rescate: El Ciclo de Vida Real del Código

¡Bienvenidos al taller! El objetivo es salvar un código defectuoso y proponer su mejora usando el flujo de trabajo real de los desarrolladores profesionales (Fork + Pull Request).

**🛠️ Instrucciones de la Parte Práctica:**

**1. Bifurcar (Fork):**
Ve a `https://github.com/LBrownI/ToolsWorkshop1` y haz clic en el botón **"Fork"** (arriba a la derecha). Esto creará una copia exacta del proyecto, pero ahora en *tu* propia cuenta de GitHub.

**2. Clonar TU copia:**
Abre la terminal en VS Code y descarga tu copia (reemplaza con tu usuario de GitHub):
`git clone https://github.com/TU_USUARIO/ToolsWorkshop1.git`

**3. Crear tu espacio y Debuguear:**

* Crea una nueva carpeta con tu nombre y apellido (ej. `lucas_brown/`).
* Copia el archivo `app.py` dentro de tu carpeta.
* Pon un *breakpoint* en la línea 4, presiona **F5** (Run & Debug) y corrige el código para que no falle con listas vacías.

**4. Guardar y Subir (Commit & Push):**
Abre la terminal de VS Code y ejecuta:
- `git add .`
- `git commit -m "fix: corregir division por cero en promedios"`
- `git push`

woa woa owa

**5. El Rescate Final (Pull Request):**
Vuelve a tu repositorio en GitHub.com. Verás un botón verde o un mensaje que dice **"Compare & pull request"** (o "Contribute"). Haz clic ahí para enviarme tu solución. ¡Acabas de hacer tu primera contribución profesional al código de otra persona!
